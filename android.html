Introduction

The Right Mindset

Every discipline on the face of the Earth requires the right mindset to create a learning and progressive atmosphere. I have learned that persistence and passion are some of the key ingredients for success. 
To be honest, I didnt know what the right mindset was in order to create this app. But I knew deep down in my heart that I wanted to create an android app. This attitude pushed me to find the 'easiest' way to make an android app, and I wasnt thinking of those low level internet services that promise to create andriod apps for you but then turn out to be nonsense. 
It wasnt until I was almost done writing this article that an epiphany hit me that the right mindset to do anything is to not give up. 

Sketchware

Tools and Skillset

How did I actually build this app? Yeah, I will explain. 
Sketchware is a platform that helps users create native android apps on your smartphone. On some website descriptions, they write "you don't need any prior programming language" but trust me, you will need to first understand the logic of computer programming before you can comfortably use this platform to create functional apps without ripping your hair out of your scalp. 

It lets you build logic programming using a new type of programming language that looks like lego blocks(I know! sounds cool right?). They have a lot of documentation and youtube video tutorials on how to arrange blocks to create awesome native apps. I will add some images of the block programming that sketchware provides. Or you can actually google it and see what it is all about. 

Aforemtioned in the introduction, I was looking for an easy way to create an app. This pet-project took about 3 weeks to complete and a few more days to perfect. It didnt cost me any money. All I had was passion and some level of imagination and creativity to actually complete this work.

I used Google Firebase database to authenticate login and sign up. By the time I was building this Android app, I had already created some webpages which will be the output.  

Smaller Problems

The bigger picture

If you have ever encountered a problem that will make you zone out, looking into the distance even in the midst of a conversation, then you will understand what I'm talking about. Even before I started creating this app, I had no idea of how the app was going to look like and that was my first real challenge, App structure. 

I took a look at many apps on google play and noticed that most of the apps that had the same functionality with what I was trying to build had "Sign up/Sign in", "Sign up form", "Main screen". This app was created using a bottom up approach. Every user has to go through a series of screens to reach the main page. 

As I continued to use Sketchware to create this project, I also discovered that this app structure would come in two parts: Layout styling and App logic. These will be explained later in this article.

Sign Up Activity

Entry Point

Every android or ios native app will always have an entry point and an exit point. The best apps will find winning strategies to keep you in the middle to continue using their app. I didn't have freedom of creativity so I used text, images and a button to create a route page to either login or sign up.

Firebase provides other login options with g-mail and facebook. Registration with email is the only option sketcware provides in its logic programming blocks. 

Each user provides a First name, Last name, Nick name, Email, Password, gender and then the registration button remains inactive unless a check box is checked. 

When the registration botton is clicked, the app checks each data point, validates it and uploads it to the firebase database. If there is no error, the user can now log in and the app automatically toasts "Registration Successful" and moves back to the login page. 

To explain computer programming can get boring very quickly, so I will not bother going through the logic of this sign-up form. Following this block of text is a series of screenshots to demonstrate the signup process.

I went back to my blueprints and I had to re-discover the idea behind this app I was trying to build. Uou must be familiar with the saying "the best ideas come to you in the bathroom", and actually, I had another "eureka" moment. As far as my entry point was complete, I could implement my main screen to be like a television to display what ever on command. 

So in the building blocks of the app, I rearranged my code to turn my main screen into a web browser and some buttons to display my web pages on the main screen. This simple yet effective approach was the key to solving my "main screen" challenge. Other elements like drawer and floating icon button add more functionality to the app.

App Drawer
UX/UI Design.

It was now getting clear the primary function of this app. "Content Delivery". It was my original intention to create an app that could help to deliver my class content to my students no matter the location. 

Design and implementation wasn't as difficult as the earlier days. I added the app drawer that contained the school logo, some navigation buttons and floating icon that prompted a dialogue box on botton press. 

The app drawer was the key feature in the app structure because it could contain many buttons that users could interact with. 

App Drawer

Exit Point 

This Android app was very exciting to build. Although the most challenging part of this project was actually build it with my mobile phone. The difficult part of the implimentation process was complete and it was time to add the finishing touches. 

After a user is finished using the app, he/she could sign out using the same authorization process provided by google firebase services. The app drawer was designed to be the navigation bar to different web pages. Each button is programmed to reload the webview (Main screen) with a new URL.

When the "User profile" button is clicked, the app returns a dialogue box with the email of the current user signed into the app. The same logic is true for "Connect Google classroom" button. So at this time, I had tested the app severally and it worked, even on other mobile devices. 

I learned that if you want to do something properly, then you will need the right tools. Sketchware is a wonderful app to build fun and interractive android apps but is limited to design and code management.

Conclusion and Summary

Fundamental Steps in Software development

Use your skills to affect your environment and the people around you. Any project, -doesnt have to always be software- will have steps or phases in its production life cycle. Apply these 5 simple steps to your problem solving strategies and increase your efficiency and accuracy in everything that you do. 
  1. Idea Phase -  Every app or software or project always has an inception stage. It could be your idea or your client's idea, but the most important lesson from this is to keep your idea as simple as pissible. If you have some resistance in generating an idea, think of a problem you can solve with a software, doesnt matter if the app already exists.
  2. Research and Methodology -  The best buildings always have beautiful and solid architecture. Before you dive head first into any project, do some research and sketch a plan on how you want to develop your idea. You will test all your ideas and imaginations on paper to see if they fit structurally and logically into your app. In this phase, I encourage you to visualise your app and think of all the colours and styling that you would love to add to your project. You can make as many mistakes as possible and you will also discover that your final product will look like an upgraded variant of your blueprint. 
  3. Implementation - Select the tools that suit your programming or development style. Before I started writing soft codes on a computer or mobile device, I used short hand psuedo code that briefly described some important processes in the app design. I also used some psuedo code in describing the way this app was built. The tools used to implement code in computer programming is very important, it can make or miss your project. An IDE (integrated development environment) will provide a code editor, build automation tools and a debugger. Choose wisely!
  4. Testing, Debugging and Troubleshooting - This phase easily merges with the implemetation phase, because as you are developing your native app, you will continue tweaking and making adjustments. It startes to get frustrating when one line of code will cause your app to crash. So in my experience, this is the time where you have to remain calm to properly debug your app because this process repeats itself in a continuous loop until there are no more errors. 
  5. Deployment- When your  project is complete, it is time to show it to the world. Many people are afraid of criticism yet that is where true growth and development stems from. At this point, after you have completed your hardwork, you will know truly for yourself the point of even trying to start and complete any project. Android apps are deployed to Google play. Well, it costs money to deploy your app to google play so I just packaged and deployed my app locally (I can make my source code available on request).

Activity States

Modern apps have states that must be true in order for something to happen. Sketchware provides components to be added to the app. 
When my sign up form is created, it sets some values and waits for user input. On back button pressed, the app prompts a dialogue box to clear the form. 

App Logic for "Register button"

In a nutshell, this block of code uses simple "if else" statements to validate each data entry point. Sketchware uses a top-down approach when running code (meaning, the IDE will read the first block before reading the next). 
Vibrator, Dialog, Timer, Intent are some of the components I used to add functionality to my form so it doesnt feel boring and redudant.

On Button Clicked

So, as a user is signing up, the app rects and responds acording to the blocks of code in the app logic. 
In summary, If the textfield for 'Nickname' is empty or the passwords dont match, gender is not selected, or user agreement isnt clicked, the form will throw an error, else, the app will save all data to the database.
